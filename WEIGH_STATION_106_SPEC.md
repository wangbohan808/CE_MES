# 称重工位（device_type=106）需求与实现说明

本文档汇总称重工位全流程中的**关键约定**，供调优、交接或让其他 AI **复现同一业务目标**时使用。与代码中搜索标记 `[WEIGH-106]`、`WEIGH-106-BEGIN/END` 可交叉对照。

---

## 1. 业务目标

在产线工位上对**已放在电子秤上的物料**完成：

1. **记录 SN**并与 MES 对接（过站/校验逻辑与现有工序一致）。
2. **获取当前重量（kg）**，与**可配置上下限**比较，得到 PASS/FAIL。
3. 将**测量值、是否合格、SN** 等与现有 `mes_run.add_report` / `send_report` 路径一致地上报（当前需求为**仅海能 MES**）。
4. **UI** 上成功/失败提示风格与 `dev >= 100` 的其他工位一致（如 `up_notification_ui`、`wx.GREEN` / `wx.RED`）。

---

## 2. 工位标识与配置

| 项 | 约定 |
|----|------|
| 设备类型 | `config.yaml` 中 `device_type: "106"` |
| 窗口标题文案 | `MainFrame.heading_line_dict["106"]`（如「称重工位」） |
| 称重上下限 | `config.yaml`：`weight_min_kg`、`weight_max_kg`（业务初值示例：**100 ~ 150**，单位 **kg**），与 `device_type` 一起随配置加载 |
| 参考区间展示 | 建议在 idle/提示区向操作员显示当前配置的 min~max（kg） |
| MES | **仅海能**（`use_mes` 等与项目现有逻辑一致；安克侧本工位不要求单独实现） |
| 海能工站码 | **正式码待 MES 确认**；代码中曾用**临时占位**，需全局搜索 `TODO` / 临时字符串并替换为正式 `UserParameter`（`station_sn_code` / `station_report_code` 与 `106` 关联项） |

---

## 3. 现场操作顺序（工艺流程）

**推荐操作顺序（与设计意图一致）：**

1. 工人将物体**预先放置在称重计上**，在现场确认为**稳定载物后再进行操作**（振动衰减后再扫码）。
2. **扫 SN**：仅此一次扫码，输入产品 SN，沿用现有「扫码进入工序」流程（MES 过站、`encode_rules` 等按项目约定执行）。
3. SN **校验通过后**：程序**间隔固定 1 秒**（用户约定：`sleep 1 s`），再向仪表发送**读重量指令**（`R\r\n`），将应答数值作为**该 SN 对应本次测量的质量**，接着按上下限判定 PASS/FAIL，并上报 MES、刷新 UI。

**不再需要**：第二次扫码触发称重；“预先放好并称相对稳定后再扫码”由操作规程保障。

---

## 4. 扫码与状态机（与其它 dev 的关系）

- **唯一一次扫码**：仍由现有主流程从 `idle` → `running`、完成单行 SN 收集（与 101 等 `dev >= 100` 的模式类似），保证与现有框架兼容。
- **读重量**：在 SN、`encode_rules`、`MES` 过站均就绪后的同一个 **`running` 周期内**延时 **1 s**，随即下发 **`R\r\n`** 并解析应答即可。
- **SN 校验**：走 `encode_rules.match_sn_encoding_rules`；用户要求**暂不强制**（当前 `encode_rules` 对未配置 dev 可为“不拦截”），但**后续**可能为 `106` 单独加规则或与 `101` 对齐，实现时应保留扩展点。

---

## 5. 串口与硬件

| 项 | 约定 |
|----|------|
| PC 连接 | **USB 转 RS232**（或等效虚拟串口 COMx） |
| 串口选择 | 与现有逻辑一致：使用 `config.yaml` 的 **`user_com`**；为空时**自动枚举**（需现场**唯一** COM，避免选错口） |
| 波特率 | **9600**（与现有工程中 101/104/105 等工位在 `use_serial.serial_open` 中的处理对齐） |
| 数据格式 | 仪表文档常见为 **8 数据位、无校验、1 停止位（8N1）**；以实际仪表菜单为准 |

### 5.1 `user_com` 为空时的自动选口（`com_id_list`）

`myserial/test_serial.py` 中 `open_test_com()`：当 **`user_com` 为空** 时，会遍历本机串口，并**优先选用** `port.pid`（PySerial 给出的 USB **产品 ID PID，十进制**）落在 **`com_id_list`** 里的设备，以便在插有多根 USB 线时尽量选对「已知芯片」的口。

| USB 硬件 ID（设备管理器 → 详细信息 → 硬件 Id） | 芯片 / 驱动 | 写入 `com_id_list` 的值 | 说明 |
|----|----|----|----|
| `USB\VID_067B&PID_2303`（及带 `&REV_...` 的变体） | **Prolific USB-to-Serial**（PL2303 等） | **`8963`** | `PID_2303` → 十六进制 `0x2303` → 十进制 **8963**，与 `ListPortInfo.pid` 一致 |
| `USB\VID_1A86&PID_7523` 等 | CH340 | `29987`（`0x7523`） | 工程内已有 |
| Silicon Labs CP210x 等 | CP2102 等 | `60000`（如 `0xEA60`） | 工程内已有 |

**现场建议**：称重工位若长期固定 COM（例如设备管理器中显示 **COM7**），最稳妥做法是在 `config.yaml` 中写死 **`user_com: "COM7"`**，不依赖自动枚举；仅在单口环境或需「优先 PL2303」时用 `com_id_list` 辅助。

**扩展新线**：若更换 USB 转串芯片，在设备管理器中查看 **VID/PID**，将 **PID 的十进制** 加入 `com_id_list`（并保持注释说明 VID+PID 字符串，便于交接）。

## 6. 仪表通讯（已验证行为）

用户用串口助手验证：

- **发送**：ASCII `R`，并勾选「**加回车换行**」（即 **`R\r\n`**）。
- **应答（HEX）**：`2B 20 32 2E 31 37 20 6B 67 20 0D 0A`
- **应答（ASCII）**：`+ 2.17 kg \r\n`（即：**符号 + 空格 + 数值 + 空格 + kg + 空格 + CRLF**）。

**模式判定**：当前现场为**命令/应答模式**（发 `R` 才回重量），不是依赖连续输出帧里 `ST,GS,...` 前缀的那一种；解析与判稳逻辑必须以**本应答格式**为准。

**不包含的需求（用户明确）：**

- 不要求在上位机实现**去皮 / 置零**流程；直接读当前返回重量即可。
- 单位按 **kg** 使用；若仪表偶发 `lb` 等，可在实现中策略性拒绝或换算（当前未强制，调优时自定）。

**手册中其它信息（供扩展）：** 连续输出可有 `ST/US/OL`、`ST,GS,+ 000.00kg\r\n` 等；若现场改模式，需同步改解析与触发逻辑。

## 7. 读数时机与延时

- **触发条件**：**第一次扫码拿到的 SN**在完成校验与海能过站通过后。
- **延时**：通过后立刻 **`sleep 1 秒`**（防抖／链路就绪／操作规程的补充）；再在 **`running`** 内发送 **`R\r\n`**。是否需要把 **1 s** 改成可配置项，可在 `config.yaml` 增加例如 `weight_read_delay_sec`。
- **发送后**：在超时窗口内从串口接收缓冲区拼出完整行并解析浮点数；超时则判为读数失败并 NG + 提示。

---

## 8. MES 与报表项

- **过站**：第一次（或按你最终代码节奏）在 SN 确定后调用与海能一致的 `check_sn_is_ok`（或项目统一入口）。
- **上报**：称重结果作为一条（或多条）测试项：`mes_run.clear_report` → `add_report`（名称、实测值、上下限、OK/NG）→ `send_report`（总结果与 SN、起止时间）。
- **本地 CSV**：沿用 `mes_run.save_report` 现有行为（与 `load_cfg.dev` 标题字典对应）。

---

## 9. UI 行为要点

- Idle 时：提示**货物先放稳再上秤、再扫 SN**（可把参考区间 min~max kg 一并展示）。
- SN 校验与过站通过后：提示**即将读数 / 正在读取重量**（例如 1 s 内）；成功绿字、失败红字，并尽量显示**实测值与配置区间**。
- 与海能通信失败时：`celink_mes` 等已有 `wx.CallAfter` 提示，避免重复刷屏可增加去重或合并文案（调优项）。

## 11. 待定 / 调优时可变项（文档级 checklist）

- [ ] 海能正式 **`UserParameter`** 替换临时占位。
- [ ] `encode_rules` 是否为 `"106"` 增加与 `101` 完全一致的强制规则。
- [ ] **第一次扫码通过后**的 **1 s** 延时是否改为 `config.yaml` 可配置（如 `weight_read_delay_sec`）。
- [ ] **等待仪表应答**的超时时长（例如 2 s）是否可调。
- [ ] 解析是否需支持 **OL/超载**、**负数**、**无小数** 等变体字符串。
- [ ] 现场若换 USB 转串芯片：在 **第 5.1 节** 对照 VID/PID 更新 `com_id_list`，或改用 **`user_com` 固定 COM**。

**实现注意**：若仓库中仍留有 **双扫码** 相关逻辑，应以**本文档本节与第 3、7 节为准**收敛为：**单次扫码 + 1 s 后发 `R\r\n`**。
